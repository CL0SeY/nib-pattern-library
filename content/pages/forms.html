We use forms to give customers the ability to provide us with information. Forms matter because the failure of form results in failure demand or a loss sale. We don't like this stuff

So, let's get forms right.

Section heading

Use the field set to represent a set of form controls e.g ADDRESS DETAILS or Payments. The headings separate sections on the page and make it easy for the user to navigate the form.

<%- partial('do-list', {items: [ 'Should clearly summarise the content of the section', 'Should be distinct from each other and unambiguous' ]}) %>
<%- partial('dont-list', {items: [ 'dummy text' ]}) %>
<%- partial('example', {source: '

ADDRESS DETAILS

'}) %>

Subsection heading

Use a sub section heading when there are multiple inputs to answer. For example radio buttons or check boxes. The sub section heading contains the "question text" in these situations.

<%- partial('example', {source: '

Home address

'}) %>

Labelling

A good label significantly helps form conversion by adding clarity without overwhelming users.

The purpose of the label is to indicate what data is required in the field. All input fields are required to have labels.

Labels should be clear and easy to understand. If the label is ambiguous, users will either guess an answer causing problems down the track, or leave the form altogether.
